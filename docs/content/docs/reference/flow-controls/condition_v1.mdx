The Condition flow control evaluates logical expressions and routes workflow execution to one of two paths: true or false. It's perfect for binary decisions like threshold checks, validation, or simple yes/no logic.

**Type**: `condition/v1`


## Properties

|      Name       |      Label     |     Type     |     Description     | Required |
|:---------------:|:--------------:|:------------:|:-------------------:|:--------:|
| rawExpression | Raw Expression | BOOLEAN <details> <summary> Options </summary> <span>true</span>, <span>true</span> </details> | Set condition as raw expression or list of conditions. | true |
| conditions | OR Conditions | ARRAY <details> <summary> Items </summary> [[&#123;STRING\(type), BOOLEAN\(value1), STRING\(operation), BOOLEAN\(value2)&#125;\($boolean), &#123;STRING\(type), DATE_TIME\(value1), STRING\(operation), DATE_TIME\(value2)&#125;\($dateTime), &#123;STRING\(type), NUMBER\(value1), STRING\(operation), NUMBER\(value2)&#125;\($number), &#123;STRING\(type), STRING\(value1), STRING\(operation), STRING\(value2), STRING\(value2)&#125;\($string)]] </details> | OR Condition array that contains AND Condition arrays | false |

## Usage

To use the Condition flow control, follow these steps in the workflow UI editor:

### Step 1: Add Condition to Your Workflow
1. Select the **Condition** component from the **Flows** component panel
2. The condition will create two paths: **True** and **False**

### Step 2: Choose Expression Type

Before moving forward, refer to this guide to learn more about [expressions](/reference/expressions). In the guide, you'll learn everything you need to know about the available operations and examples. 

The **Raw Expression** toggle determines how you define your condition:
#### If Raw Expression is **TRUE** (enabled):
You write a direct expression in the **Expression** field:
- Example: `${order_1.total} > 1000` (Ensure data expressions like `{order_1.total}` are referenced from the data pills)
- Example: `${user_1.verified} == true && ${user_1.age} >= 18`

Note that when working with certain expressions, you may need to refer to the data pills available in the workflow context. Data pills may be based on data generated from components or triggers or from a manually configured source in a component.

#### If Raw Expression is **FALSE**:
You use the visual condition builder with **OR Conditions** that contain **AND Condition** arrays:

1. Click **Add OR Condition** to create a new OR group
2. Within each OR group, click **Add AND Condition** to add conditions
3. Select expression type for each condition:
   - **Boolean Expression** - True/false comparisons
   - **Date Time Expression** - Date/time comparisons
   - **Number Expression** - Numeric comparisons
   - **String Expression** - Text comparisons

### Step 3: Configure Your Condition

**Using Raw Expression (Raw Expression = TRUE):**
```
Example Expression: ${order_1.total} > 1000 && ${order_1.status} == "pending"
```

**Using Condition Builder (Raw Expression = FALSE):**

### Example: Building `${order_1.total} > 1000 && ${order_1.status} == "pending"`

To build this expression using the Condition Builder:

1. **Set Raw Expression to FALSE** to use the visual builder
2. **Add OR Condition** - Click to create the first OR group
3. **Add AND Conditions** within the OR group:
   - **First AND Condition:**
     - Type: **Number Expression**
     - Value 1: `${order_1.total}` (select from data pills)
     - Operation: **Greater**
     - Value 2: `1000`
   
   - **Second AND Condition:**
     - Type: **String Expression**
     - Value 1: `${order_1.status}` (select from data pills)
     - Operation: **Equals**
     - Value 2: `"pending"`

4. **Result**: The condition builder will automatically generate: `${order_1.total} > 1000 && ${order_1.status} == "pending"`

### Step 4: Build True and False Paths
1. Add components to the **True** path for when condition evaluates to true
2. Add components to the **False** path for when it evaluates to false

You can refer to this [json file](https://www.npoint.io/docs/73d362abb3b155274638) to see the Condition flow control in action.
