## Actions


### Cancel Order
Name: cancelOrder

`Cancels an order, with options for refunding, restocking inventory, and customer notification.Order cancellation is irreversible.`

#### Properties

|      Name       |      Label     |     Type     |     Description     | Required |
|:---------------:|:--------------:|:------------:|:-------------------:|:--------:|
| orderId | Order ID | STRING | ID of the order to cancel. | true |
| reason | Reason | STRING <details> <summary> Options </summary> <span>CUSTOMER</span>, <span>DECLINED</span>, <span>FRAUD</span>, <span>INVENTORY</span>, <span>OTHER</span>, <span>STAFF</span> </details> | The reason for canceling the order. | true |
| restock | Restock | BOOLEAN <details> <summary> Options </summary> <span>true</span>, <span>false</span> </details> | Whether to restock the inventory committed to the order. | true |
| staffNote | Staff Note | STRING | A staff-facing note about the order cancellation. This is not visible to the customer. | false |
| originalPaymentMethodsRefund | Original Payment Methods Refund | BOOLEAN <details> <summary> Options </summary> <span>true</span>, <span>false</span> </details> | Whether to refund to the original payment method. | false |
| notifyCustomer | Notify Customer | BOOLEAN <details> <summary> Options </summary> <span>true</span>, <span>false</span> </details> | Whether to send a notification to the customer about the order cancellation. | false |

#### Example JSON Structure
```json
{
  "label" : "Cancel Order",
  "name" : "cancelOrder",
  "parameters" : {
    "orderId" : "",
    "reason" : "",
    "restock" : false,
    "staffNote" : "",
    "originalPaymentMethodsRefund" : false,
    "notifyCustomer" : false
  },
  "type" : "shopify/v1/cancelOrder"
}
```

#### Output



Type: OBJECT


#### Properties

|     Name     |     Type     |     Description     |
|:------------:|:------------:|:-------------------:|
| job | OBJECT <details> <summary> Properties </summary> &#123;BOOLEAN\(done), STRING\(id), STRING\(query)&#125; </details> | The job that asynchronously cancels the order. |
| userErrors | ARRAY <details> <summary> Items </summary> [&#123;STRING\(field), STRING\(message)&#125;] </details> | The list of errors that occurred from executing the mutation. |




#### Output Example
```json
{
  "job" : {
    "done" : false,
    "id" : "",
    "query" : ""
  },
  "userErrors" : [ {
    "field" : "",
    "message" : ""
  } ]
}
```

# Additional Instructions

## Connection Setup

### Create Shopify App (Credentials and Shop Name)

1. Navigate to https://dev.shopify.com/dashboard/.
2. Click on **Create app**.
3. Enter App name and click on **Create**.
4. Enter app and redirect url. Depending on which version of ByteChef you are using enter link or link for **Redirect URL**. And link or link for **App URL**.
5. Enter required scopes:
    read_customers,read_draft_orders,read_orders,write_orders,write_products
and click on **Release**
6. Click on **Release**.
7. Click on **Settings**.
8. Here you can see your credentials.
9. Click on your account icon.
10. Click on shop you want to connect to.
11. Click on **Settings**.
12. Here you can see your **Shop name**.
13. Done ðŸš€.

1. Navigate to https://dev.shopify.com/dashboard/.

5. Click on **Install**.

### Enable Read All Orders Permission

1. Navigate to https://partners.shopify.com/organizations.
2. Choose your organization.
3. Click on your app.
4. Click on **API access requests**.
5. Click on **Choose distribution**.
6. Click on **Custom Distribution**.
7. Click on **Select**.
8. Click on **Select custom distribution**.
9. Enter your **Shop name** here. In this example it is `bytechef-test-store.myshopify.com`.
10. Click on **Generate link**.
11. Click on **Generate link**.
12. Click on **API access requests**.
13. Click on **Request access** under **Read all orders scope**.
14. Enter reason here.
15. Click on **Request access**.
16. Navigate to https://dev.shopify.com/dashboard/.
17. Click on your app.
18. Click on **Versions**.
19. Click **Create a version**.
20. Add `read_all_orders` to scopes.
21. Click on **Release**.
22. Click on **Release**.
23. Done ðŸš€.
