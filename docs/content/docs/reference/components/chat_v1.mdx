## Actions


### Response to Chat Request
Name: responseToRequest

`Converts the response to chat request.`

#### Properties

|      Name       |      Label     |     Type     |     Description     | Required |
|:---------------:|:--------------:|:------------:|:-------------------:|:--------:|
| message | Message | STRING | The message of the response. | false |
| attachments | Attachments | ARRAY <details> <summary> Items </summary> [FILE_ENTRY] </details> | The attachments of the response. | false |

#### Example JSON Structure
```json
{
  "label" : "Response to Chat Request",
  "name" : "responseToRequest",
  "parameters" : {
    "message" : "",
    "attachments" : [ {
      "extension" : "",
      "mimeType" : "",
      "name" : "",
      "url" : ""
    } ]
  },
  "type" : "chat/v1/responseToRequest"
}
```

#### Output

The output for this action is dynamic and may vary depending on the input parameters. To determine the exact structure of the output, you need to execute the action.





## Triggers


### New Chat Request
Name: newChatRequest

`A new chat request comes from the chat interface.`

Type: STATIC_WEBHOOK

#### Properties

|      Name       |      Label     |     Type     |     Description     | Required |
|:---------------:|:--------------:|:------------:|:-------------------:|:--------:|
| mode | | INTEGER <details> <summary> Options </summary> <span title="Use ByteChef's hosted chat interface">1</span>, <span title="This option requires you to create your own chat interface">2</span> </details> |  | true |


#### Output



Type: OBJECT


#### Properties

|     Name     |     Type     |     Description     |
|:------------:|:------------:|:-------------------:|
| conversationId | STRING |  |
| message | STRING |  |
| attachments | ARRAY <details> <summary> Items </summary> [FILE_ENTRY] </details> |  |




#### JSON Example
```json
{
  "label" : "New Chat Request",
  "name" : "newChatRequest",
  "parameters" : {
    "mode" : 1
  },
  "type" : "chat/v1/newChatRequest"
}
```
