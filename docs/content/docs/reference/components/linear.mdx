## Actions


### Create Issue
Name: createIssue

`Creates a new issue.`

#### Properties

|      Name       |      Label     |     Type     |     Description     | Required |
|:---------------:|:--------------:|:------------:|:-------------------:|:--------:|
| title | Issue Title | STRING | `The title of the new issue.` | true |
| teamId | Team ID | STRING | `The ID of the team where this issue should be created.` | true |
| statusId | Status | STRING | `The status of the issue.` | true |
| priority | Priority | INTEGER <details> <summary> Options </summary> 0, 1, 2, 3, 4 </details> | `The priority of the issue.` | false |
| assigneeId | Assignee ID | STRING | `The identifier of the user to assign the issue to.` | false |
| description | Description | STRING | `The detailed description of the issue.` | false |

#### Example JSON Structure
```json
{
  "label" : "Create Issue",
  "name" : "createIssue",
  "parameters" : {
    "title" : "",
    "teamId" : "",
    "statusId" : "",
    "priority" : 1,
    "assigneeId" : "",
    "description" : ""
  },
  "type" : "linear/v1/createIssue"
}
```

#### Output



Type: OBJECT


#### Properties

|     Name     |     Type     |     Description     |
|:------------:|:------------:|:-------------------:|
| success | BOOLEAN <details> <summary> Options </summary> true, false </details> | Whether the operation was successful. |
| issue | OBJECT <details> <summary> Properties </summary> &#123;STRING\(id), STRING\(title)&#125; </details> | The issue that was created or updated. |




#### Output Example
```json
{
  "success" : false,
  "issue" : {
    "id" : "",
    "title" : ""
  }
}
```

## Triggers


### New Issue
Name: newIssue

`Triggers when new issue is created.`

Type: DYNAMIC_WEBHOOK

#### Properties

|      Name       |      Label     |     Type     |     Description     | Required |
|:---------------:|:--------------:|:------------:|:-------------------:|:--------:|
| allPublicTeams | All Public Teams | BOOLEAN <details> <summary> Options </summary> true, false </details> | `If true, the webhook will be created for all public teams.` | true |
| teamId | Team ID | STRING |  | true |


#### Output



Type: OBJECT


#### Properties

|     Name     |     Type     |     Description     |
|:------------:|:------------:|:-------------------:|
| id | STRING | The unique identifier of the entity. |
| title | STRING | The issue's title. |
| team | OBJECT <details> <summary> Properties </summary> &#123;STRING\(id), STRING\(name)&#125; </details> | The team that the issue is associated with. |
| state | OBJECT <details> <summary> Properties </summary> &#123;STRING\(name)&#125; </details> | The workflow state that the issue is associated with. |
| priority | STRING | The priority of the issue. |
| assignee | OBJECT <details> <summary> Properties </summary> &#123;STRING\(id), STRING\(name)&#125; </details> | The user to whom the issue is assigned to. |
| description | STRING | The issue's description in markdown format. |




#### JSON Example
```json
{
  "label" : "New Issue",
  "name" : "newIssue",
  "parameters" : {
    "allPublicTeams" : false,
    "teamId" : ""
  },
  "type" : "linear/v1/newIssue"
}
```
