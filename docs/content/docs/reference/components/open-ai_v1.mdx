### Create Image
Name: createImage

`Create an image using text-to-image models`

#### Properties

|      Name       |      Label     |     Type     |     Description     | Required |
|:---------------:|:--------------:|:------------:|:-------------------:|:--------:|
| model | Model | STRING <details> <summary> Options </summary> <span>dall-e-2</span>, <span>dall-e-3</span> </details> | The model to use for image generation. | true |
| imageMessages | Messages | ARRAY <details> <summary> Items </summary> [&#123;STRING\(content), NUMBER\(weight)&#125;] </details> | A list of messages comprising the conversation so far. | true |
| size | Size | STRING <details> <summary> Options </summary> <span>DALL_E_2_256x256</span>, <span>DALL_E_2_512x512</span>, <span>_1024x1024</span>, <span>DALL_E_3_1792x1024</span>, <span>DALL_E_3_1024x1792</span> </details> | The size of the generated images. | true |
| n | Number of Responses | INTEGER | The number of images to generate. Must be between 1 and 10. For dall-e-3, only n=1 is supported.. | false |
| responseFormat | Response format | STRING <details> <summary> Options </summary> <span>URL</span>, <span>B64_JSON</span> </details> | The format in which the generated images are returned. | false |
| quality | Quality | STRING <details> <summary> Options </summary> <span>STANDARD</span>, <span>HD</span> </details> | The quality of the image that will be generated. | false |
| style | Style | STRING <details> <summary> Options </summary> <span>VIVID</span>, <span>NATURAL</span> </details> | The style of the generated images. Must be one of vivid or natural. Vivid causes the model to lean towards generating hyper-real and dramatic images. Natural causes the model to produce more natural, less hyper-real looking images. This parameter is only supported for dall-e-3. | false |
| user | User | STRING | A unique identifier representing your end-user, which can help admins to monitor and detect abuse. | false |

# Additional Instructions

## Connection Setup

1. Go to the [OpenAI API](https://platform.openai.com/settings/organization/general).
2. In the left sidebar, click on **API Keys**.
3. Click **Create new secret key**.
4. Enter a name and select the project for the key and click **Create secret key**.
5. Copy the API key and use it to create a connection in ByteChef.
