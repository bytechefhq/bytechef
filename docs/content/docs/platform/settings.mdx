# Settings

Open **Settings** from your user menu (most areas require administrator privileges). This page maps every settings area to what it controls and where to read more.

## Workspace & organization

| Settings area | What it controls |
|---|---|
| **Workspaces** | Create and manage [workspaces](/enterprise/collaboration-devops/workspaces-projects) — the grouping for projects, connections, data tables, and knowledge bases. |
| **Users** | Workspace members and their [roles](/enterprise/governance-security/rbac) (Admin / Editor / Viewer); invite and remove members. |
| **Connections** | Organization-level view of shared connections — see [Connection Visibility](/enterprise/governance-security/connection-visibility) *(coming soon)*. |
| **Git Configuration** | The Git repository and credentials behind [Git-backed change tracking](/enterprise/collaboration-devops/git-backed-change-tracking). |

## API access

| Settings area | What it controls |
|---|---|
| **API Keys** (workspace) | Keys for the automation public API, bound to a workspace and environment. |
| **Admin API Keys** | Instance-level admin keys — also the credential for the [management MCP server](/platform/mcp-server) *(coming soon)*. |
| **API Connectors** | Generate a new connector from an OpenAPI specification, build one manually, or let AI generate it — the resulting connector appears as a component. |
| **OAuth2 Clients** | Clients registered against the embedded OAuth2 authorization server (including dynamically registered MCP clients) — list and revoke them *(coming soon)*. |

## AI

| Settings area | What it controls |
|---|---|
| **AI Providers** | The central [provider catalog](/platform/ai-providers) — activate providers and enter API keys per environment. |
| **MCP Server** | The [management MCP server](/platform/mcp-server) URL and secret *(coming soon)*. |
| **AI Hub Connectors** | Which connectors the [AI Hub](/automation/ai-hub) assistant can use in conversations *(coming soon)*. |

## Governance & extensibility

| Settings area | What it controls |
|---|---|
| **Component Policies** | Which [components builders can see and run](/enterprise/governance-security/component-policies), toggled tenant-wide *(coming soon)*. |
| **Custom Components** | Upload and manage [custom components](/enterprise/extensibility/custom-components). |
| **Identity Providers** | External IdP registrations for [Enterprise SSO](/enterprise/governance-security/sso) *(coming soon)*. |
| **Audit Events** | The [audit log](/enterprise/governance-security/audit-log) of privileged actions *(coming soon)*. |
| **License** | Activate and inspect your [Enterprise license](/enterprise/governance-security/license-gated-distribution). |
| **Notifications** | Where ByteChef sends operational notifications *(coming soon)*. |

## Your account

Personal preferences live under **Account**:

- **Profile** — name, email, password.
- **Appearance** — light / dark theme.
- **Active Sessions** — review and revoke your signed-in sessions.
