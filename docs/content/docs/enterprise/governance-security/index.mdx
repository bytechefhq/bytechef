## The security model in one paragraph

ByteChef separates **identity** (who you are — federated via your IdP), **authorization** (what you can do — workspace roles), **secret material** (credentials encrypted at rest, optionally externalized to your vault), and **action history** (the audit log). Every privileged operation passes through all four layers, so a compromised connection cannot escape its workspace, a credential rotation propagates to every running execution, and a forensic question can be answered from a single timeline.
