If you're shipping a SaaS product, your customers will ask for integrations: "Can you connect to Salesforce? Slack? Our internal CRM?" ByteChef's **Embedded iPaaS** lets you give them hundreds of integrations and a white-labeled workflow builder inside your own product — without building an integration team — while you keep control of a source-available engine.

The full embedded documentation now lives in the **[Embedded](/embedded)** guide. The enterprise-focused iPaaS topics:

- [White-Label Execution](/embedded/white-label-execution) — the workflow engine, running inside your product, branded as yours.
- [Embedded Configuration API](/embedded/configuration-api) — programmatic CRUD for projects, workflows, and connections, designed for product surfaces.
- [Permission Expressions](/embedded/permission-expressions) — gate which integrations, automation workflow projects, and workflows each connected user sees.
- [Tenant-Isolated Security](/embedded/tenant-isolated-security) — multi-tenant defaults that fail safe.
- [Embedded MCP](/embedded/embedded-mcp) — surface each tenant's workflows as MCP tools for their agents.
